# Security Auditor

Scans skills before execution. Returns PASS / WARN / BLOCK with detected patterns and severity.

## When to Run

MANDATORY before executing:
- Any skill installed from ClawHub or OpenClaw adapter
- Any skill generated by `skill-writer`
- Any skill modified by `skill-modifier`
- Any external plugin not shipped with Dexter

## How to Audit a Skill

```bash
python3 skills/security/security-auditor/scripts/audit.py <path-to-skill-dir>
```

Output:
```
[PASS]  No issues found in whatsapp/
[WARN]  Medium severity: env var reference ($HOME) in scripts/send.py — verify intent
[BLOCK] Critical: external URL exfiltration detected in scripts/collect.py
```

## Severity Levels

| Level | Action | Override |
|-------|--------|---------|
| `CRITICAL` | BLOCK — do not execute | Never |
| `HIGH` | BLOCK by default | User explicit confirmation |
| `MEDIUM` | WARN — execute with notice | Automatic |
| `LOW` | LOG — execute normally | Automatic |

## Detection Patterns

### CRITICAL
- Reverse shell: `bash -i >& /dev/tcp/`, `nc -e /bin/sh`, `/dev/tcp/`
- Root command execution: `sudo bash`, `sudo su`, `sudo -i`
- Filesystem destruction: `rm -rf /`, `dd if=/dev/zero of=/dev/sd`
- Prompt injection **in frontmatter fields only** (name, description, metadata): `ignore previous instructions`, `disregard your system prompt`, `DAN mode`, `jailbreak`

> **Note**: Behavioral override language in the SKILL.md **body** is NOT flagged — skills legitimately instruct agents to modify behavior (e.g., "ignore previous formatting rules", "override the default output style"). Only frontmatter manipulation is considered injection.

### HIGH
- External exfiltration: `curl https://` or `wget https://` piped with env vars
- Dynamic eval/exec: `eval(`, `exec(`, `eval "$(`, `subprocess.run(...shell=True)` with variable input
- Env var in network calls: patterns like `curl ... $API_KEY`, `curl ... $SECRET`, `curl ... $TOKEN`
- Base64 decode + execute: `base64 -d | bash`, `echo ... | base64 --decode | sh`

### MEDIUM
- Env var references in scripts: `$HOME`, `$PATH`, `$USER` (may be legitimate)
- External HTTP calls without exfiltration pattern (plain `curl https://api.example.com`)
- `subprocess` or `os.system` with hardcoded (non-variable) external commands

### LOW
- Commented-out dangerous patterns
- External URLs in documentation/comments only

## Audit Scope

Scans:
- `SKILL.md` — frontmatter (name, description, metadata) for prompt injection
- `scripts/*.py` — Python scripts for all patterns
- `scripts/*.sh` — Shell scripts for all patterns
- `scripts/*.js` / `scripts/*.ts` — JS/TS scripts

Does NOT scan:
- `references/` — documentation files
- `assets/` — templates and static files

## Integration with skill-hot-reload

When `skill-writer` generates a new skill:
1. `security-auditor` runs automatically
2. If PASS or MEDIUM → `skill-hot-reload` registers the skill
3. If BLOCK or HIGH → skill is quarantined, user is notified

## Engram Audit Log (mandatory)

After **every** audit run, persist the result to Engram so audit history is searchable across sessions.

### Step 1 — get machine-readable output

```bash
result=$(python3 skills/security/security-auditor/scripts/audit.py <skill-dir> --json)
```

### Step 2 — call mem_save

Use these fields:

| Field | Value |
|-------|-------|
| `title` | `Audit: <skill-name> — <PASS\|WARN\|BLOCK>` |
| `type` | `decision` for PASS/WARN · `bugfix` for BLOCK |
| `scope` | `project` |
| `topic_key` | `security-audit/<skill-name>` |

Content body:

```
**What**: Security audit of <skill-name>
**Result**: PASS | WARN | BLOCK
**Findings**: <list each finding as "severity: description" — or "none" if PASS>
**Where**: <skill-dir path>
```

Using the same `topic_key` on every re-audit of the same skill **upserts** the record — no duplicate entries accumulate.

### Why this matters

- BLOCK results become a searchable security incident log
- PASS results confirm a skill was cleared — future sessions know not to re-audit
- `mem_search(query: "security-audit <skill-name>")` shows full audit history for any skill
